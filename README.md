# SeedLM: Semantic-Importance-Ordered Text Generation

意味的重要度に基づく階層的テキスト成長型生成の実験リポジトリ

## Overview

従来の言語モデルは左から右へ逐次的にテキストを生成する（autoregressive）。
本プロジェクトでは、**文の意味的核（seed）から外側へ有機的にテキストを成長させる**という新しい生成パラダイムを探索する。

```
step0: 美味しい
step1: 美味しいです。
step2: が美味しいです。
step3: コーヒーが美味しいです。
step4: 挽いたコーヒーが美味しいです。
step5: タイムモアで挽いたコーヒーが美味しいです。
```

画像生成における低解像度→高解像度のプロセスに似ているが、テキストは離散的（UTF-8）であるため連続空間の補間ができない。
本アプローチでは、意味の「種」から構文的・修飾的要素を段階的に付加していくことで、この課題に対する新しい解を提案する。

## Motivation

### なぜ左から右でなければならないのか？

人間が文章を考えるとき、必ずしも先頭から順番に組み立てるわけではない。
「コーヒーが美味しい」と伝えたいとき、まず「美味しい」という核があり、そこに「何が？」→「コーヒー」、「どんな？」→「挽いた」と詳細が付加されていく。

小説も同様で、一番伝えたいテーマ（seed）を核として、それを補完・拡張していく営みである。
step を無限にすれば小説になり、step を 1 にすればキーワードになる。

### 日本語の言語的優位性

日本語の助詞（は、が、を、で、に）は独立したトークンとして文法的役割を明示する。
これにより「構文的骨格」と「意味的内容語」を明確に分離した実験が可能になる。
英語は語順に依存するため、文法を先に生成する実験が本質的に難しい。

## 本プロジェクトの差別化ポイント

既存研究との差分を明確にする。

| 観点         | 既存研究                           | SeedLM                                     |
| ------------ | ---------------------------------- | ------------------------------------------ |
| 生成方式     | 固定長の穴埋め or 挿入             | テキスト自体が成長（出力長が動的に決まる） |
| 重要度の基準 | TF-IDF / モデル確信度 / 構文的位置 | 意味的重要度（情報量ベース）               |
| 対象言語     | ほぼ全て英語                       | 日本語（助詞構造の活用）                   |
| 動機         | 高速化 / 制約付き生成              | 人間の思考過程に近い生成                   |
| 粒度制御     | なし（固定 step）                  | step 数による連続的な詳細度制御            |

## Experiments

学習なし（推論のみ）で実施。モデルは `cl-tohoku/bert-base-japanese-whole-word-masking` を使用。

### Experiment 1: BERT 品詞別アンマスク順序分析

全マスク状態からBERTがどの品詞を先に復元するかを観察。

**結果**: BERTは構文的骨格（助詞）を最初に復元する。

| 品詞   | 平均アンマスクステップ | 分類   |
| ------ | ---------------------- | ------ |
| 助詞   | 4.5                    | 機能語 |
| 動詞   | 6.4                    | 内容語 |
| 名詞   | 6.7                    | 内容語 |
| 助動詞 | 7.2                    | 機能語 |
| 形容詞 | 8.7                    | 内容語 |
| 副詞   | 12.0                   | 内容語 |

Ford et al. (2018) の「機能語→内容語の順が最適」という報告を日本語の助詞構造で独立に確認。BERTにとって自然な順序は「構文→意味」であり、SeedLMが目指す「意味→構文」とは真逆であることが判明。

### Experiment 1b: 方向別マスク補完精度

内容語と機能語、どちらを手がかりとして与えた方がBERTは補完できるかを検証。

**結果**: 圧倒的な非対称性を確認。

| パターン             | 精度         | 平均確信度 |
| -------------------- | ------------ | ---------- |
| A: 内容語→機能語補完 | 21.9% (7/32) | 0.1166     |
| B: 機能語→内容語補完 | 0.0% (0/28)  | 0.0017     |

確信度に約70倍の差。内容語が与えられれば助詞をある程度補完できるが、助詞だけでは内容語はほぼ予測不能（最頻出名詞に崩壊）。SeedLMの「意味の核から構文を補完する」方向が、逆方向よりもはるかに実現可能性が高いことを示唆。

### Experiment 1c: 反復的機能語補完

一斉マスクではなく、確信度順に1語ずつアンマスクする反復方式で精度を検証。

**結果**: 反復的アンマスクで精度が2.5倍以上に向上。

| 方式             | 精度          | 改善幅   |
| ---------------- | ------------- | -------- |
| 一斉マスク       | 21.9% (7/32)  | baseline |
| 反復的アンマスク | 56.2% (18/32) | +34.4pt  |

精度推移にも明確なパターンが存在：

| 進捗           | 精度   | 解釈                                              |
| -------------- | ------ | ------------------------------------------------- |
| 序盤 (0-25%)   | 50%    | 高確信度の助詞から着手                            |
| 中盤 (26-75%)  | 75-86% | 文脈が揃うほど精度が急上昇                        |
| 終盤 (76-100%) | 27%    | 「です/ます」等の文体選択は意味的制約で決まらない |

終盤の精度低下は「です/ます」vs「だ/である」のような文体レベルの選択であり、意味的制約だけでは決定できないもの。これは欠点ではなく、意味レイヤーと文体レイヤーの自然な分離を示している。

### 暫定的な生成レイヤーモデル

実験結果から、以下の3層構造が浮かび上がった：

```
Layer 1: 意味の核（内容語）     ← 人間 or LLM が生成
Layer 2: 文法的接続（助詞）     ← BERT で 56-86% 補完可能
Layer 3: 文体選択（です/ます）  ← パラメータで制御
```

### Experiment 2: LLaDA アンマスク可視化（暫定）

拡散型LLM（LLaDA-8B-Instruct）の推論過程で、確信度順アンマスクにおける品詞別の順序を観察。Google Colab A100 で実施。

**結果**: LLaDA は BERT と逆の順序でアンマスクする。

| モデル       | 内容語の平均正規化ステップ | 機能語の平均正規化ステップ | 傾向       |
| ------------ | -------------------------- | -------------------------- | ---------- |
| BERT (Exp1)  | 0.60（遅い）               | 0.38（早い）               | 機能語が先 |
| LLaDA (Exp2) | 0.30（早い）               | 0.55（遅い）               | 内容語が先 |

品詞別の詳細（LLaDA、正規化ステップ 0=最初, 1=最後）：

| 品詞   | 平均正規化ステップ | 分類   |
| ------ | ------------------ | ------ |
| 名詞   | 0.047              | 内容語 |
| 助詞   | 0.088              | 機能語 |
| 形容詞 | 0.914              | 内容語 |
| 助動詞 | 0.948              | 機能語 |

名詞・助詞が序盤に確定し、形容詞・助動詞（文体要素）が終盤に確定するパターン。BERTの「助詞→名詞→形容詞」とは対照的に、LLaDAは「名詞→助詞→形容詞→助動詞」の順で、意味の核から外側へ成長する傾向を示す。

**注意**: 暫定データ（有効33トークン、5プロンプト中3つで生成品質に問題あり）。バイトレベルBPEトークンの品詞アライメント精度にも改善の余地がある。傾向の確認には追実験が必要。

### BERT vs LLaDA の対比

```
BERT:  助詞(0.38) → 動詞(0.53) → 名詞(0.56) → 助動詞(0.60) → 形容詞(0.73) → 副詞(1.0)
       ^^^^^^^^^^^^^^^^                                        ^^^^^^^^^^^^^^^^^^^^^^^^
       機能語が先に確定                                          内容語が後

LLaDA: 名詞(0.05) → 助詞(0.09) → ........... → 形容詞(0.91) → 助動詞(0.95)
       ^^^^^^^^^^^^^^^^                         ^^^^^^^^^^^^^^^^^^^^^^^^^^
       内容語が先に確定                           修飾・文体要素が後
```

BERTは構文骨格を先に構築してから意味を埋める（Ford et al. 2018 と整合）。LLaDAは意味の核を先に確定してから構文・修飾を補完する。この違いは、マスク言語モデル（BERT）と拡散型言語モデル（LLaDA）の学習方式の差に起因する可能性がある。

**SeedLM への示唆**: 拡散型モデルは、明示的に指示しなくても「意味の核→構文→修飾」の順序を自然に獲得している。これは SeedLM の仮説——意味的核から外側へテキストを成長させることが可能——を支持する証拠である。

### Experiment 3: Seed からの段階的テキスト成長

Exp1〜2 で「既存モデルがどの順序でトークンを復元するか」を分析した。Exp3 では「seed 語から実際にテキストを成長させる」合成フェーズに移行。Exp1c で発見した 3 層モデル（意味の核→文法的接続→修飾・文体）を検証する。

2 つのモードで実験：
- **Mode A（目標文ガイド付き）**: 目標文中の seed 語以外を全マスク → 確信度順に 1 語ずつ復元（oracle mode）
- **Mode B（自由成長）**: seed 語のみから `[MASK]` 挿入 → BERT 予測 → 確信度閾値で受理 → 繰り返し

#### Mode A の成長例

「この本はとても面白かったので友達に勧めました。」（seed: 面白かった）

```
step 0: ■ ■ ■ ■ 面白かった ■ ■ ■ ■ ■ ■ ■ ■        [seed]
step 1: ■ 本 ■ ■ 面白かった ■ ■ ■ ■ ■ ■ ■ ■        → 名詞(L1)
step 2: ■ 本 ■ ■ 面白かった ■ ■ 友達 ■ ■ ■ ■ ■      → 名詞(L1)
step 3: ■ 本 ■ ■ 面白かった ■ ■ 友達 に ■ ■ ■ ■      → 助詞(L2)
step 4: ■ 本 は ■ 面白かった ■ ■ 友達 に ■ ■ ■ ■      → 助詞(L2)
 ...
step 8: この 本 は ■ 面白かった の で 友達 に ■ ■ ■ 。   → 補助記号(L3)
step 9: この 本 は ■ 面白かった の で 友達 に ■ ■ た 。   → 助動詞(L3)
step10: この 本 は とても 面白かった の で 友達 に ■ ■ た 。 → 副詞(L3)
step11: この 本 は とても 面白かった の で 友達 に 勧め ■ た 。 → 動詞(L1)
step12: この 本 は とても 面白かった の で 友達 に 勧め まし た 。 → 助動詞(L3)
```

意味の核（本、友達）→ 文法的接続（に、は）→ 修飾・文体（。、た、とても）の順で成長。3 層モデルの予測と整合。

#### 3 層モデルの検証結果（全 5 文集計）

| 層 | 平均正規化ステップ | 精度 | n |
|---|---|---|---|
| Layer 1: 意味の核（名詞・動詞） | 0.495 | 15.0% | 20 |
| Layer 2: 文法的接続（助詞・接続詞） | 0.379 | 5.6% | 18 |
| Layer 3: 修飾・文体（助動詞・形容詞・副詞・記号） | 0.596 | 40.0% | 15 |

文ごとの 3 層順序：

| 文 | L1 | L2 | L3 | L1<L2<L3 |
|---|---|---|---|---|
| タイムモアで挽いたコーヒーが美味しいです。 | 0.750 | 0.200 | 0.367 | ✗ |
| 東京の桜は春に最も美しく咲きます。 | 0.333 | 0.364 | 0.727 | ✓ |
| 彼女は毎朝公園でジョギングをしています。 | 0.396 | 0.533 | 0.625 | ✓ |
| この本はとても面白かったので友達に勧めました。 | 0.333 | 0.357 | 0.629 | ✓ |
| 雨が降っているから傘を持っていきなさい。 | 0.551 | 0.308 | 0.538 | ✗ |

**5 文中 3 文**で L1 < L2 < L3 の完全な 3 層順序が確認された。集計でも L1(0.495) < L3(0.596) は成立。ただし L2(0.379) が L1 より早い — これは BERT の助詞優先傾向（Exp1 と整合）によるもの。

#### Exp1（全マスク復元）との比較

seed 語の存在が復元精度にどう影響するかを、同じ文で Exp1（全マスク→確信度順復元）と比較。

| 文 | Exp1 精度 | Exp3 精度 | 差分 |
|---|---|---|---|
| タイムモアで挽いたコーヒーが美味しいです。 | 0.0% | 0.0% | +0.0pt |
| 東京の桜は春に最も美しく咲きます。 | 0.0% | 20.0% | +20.0pt |
| 彼女は毎朝公園でジョギングをしています。 | 0.0% | 18.2% | +18.2pt |
| この本はとても面白かったので友達に勧めました。 | 21.4% | 25.0% | +3.6pt |
| 雨が降っているから傘を持っていきなさい。 | 15.4% | 27.3% | +11.9pt |
| **全体** | **8.3%** | **18.9%** | **+10.6pt** |

seed 語が 1 つ見えているだけで予測精度が大幅に向上（+10.6pt）。特に Exp1 で精度 0% だった文が seed 付きで 18〜20% に改善しており、**意味の核が手がかりとして機能する**ことを示す。

#### Mode B: 自由成長

seed 語のみから目標文なしで成長を試みる探索実験。

```
step 0: 美味しい                               [seed]
step 1: 美味しい 社会                            → 確信度: 0.19
step 2: 美味しい 社会 社会                        → 確信度: 0.85
step 3: 停止（反復トークン検出: 「社会」が3回連続）
```

BERT は穴埋めモデルであり、文脈のない `[MASK]` に対しては最頻出トークン（「社会」等）に崩壊する。**自由成長には BERT 単体では不十分**であり、拡散型モデル（LLaDA 等）や外部からの構造制約が必要。

#### Exp3 のまとめ

1. **3 層モデルは部分的に支持された**: L1 → L3 の大局的順序は確認。ただし L2（助詞）は BERT の性質上 L1 より早く復元される
2. **seed は有効な手がかり**: 全マスク比で精度 +10.6pt。意味の核から構文を補完する方向性は実現可能
3. **BERT の限界**: 自由成長（Mode B）では崩壊。BERT は「与えられた枠を埋める」モデルであり、「枠自体を生成する」能力はない
4. **次のステップ**: 拡散型モデル（LLaDA）で同様の seed 付き成長実験を行い、BERT との違いを比較する

### 今後の実験

- **Experiment 4**: 拡散型モデル（LLaDA）での seed 付き成長 — BERT では限界があった自由成長が、拡散型モデルで改善されるかを検証
